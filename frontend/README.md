# Messenger Frontend

Frontend - клиентская часть мессенджера на React, TypeScript и Vite. Интерфейс включает авторизацию, список чатов, контакты, профиль пользователя, окно переписки, поиск по сообщениям, вложения и переключение темы.

## Стек

- React 19
- TypeScript
- Vite
- ESLint
- CSS Modules не используются, стили лежат в обычных CSS-файлах

## Основные файлы

```text
src/
├── api/index.ts              # HTTP API, WebSocket-клиент, типы данных
├── App.tsx                   # состояние приложения и сборка экранов
├── App.css                   # основные стили интерфейса
├── components/               # панели, формы, чат, сообщения, профиль
├── hooks/useTheme.ts         # переключение темы
└── utils/                    # работа с аватарами, чатами и ссылками на файлы
```

## Возможности интерфейса

- регистрация и вход;
- сохранение JWT и пользователя в `localStorage`;
- просмотр и создание чатов;
- создание личного диалога с контактом;
- добавление и удаление друзей;
- редактирование профиля и аватара;
- отправка сообщений через WebSocket;
- загрузка файлов и изображений;
- редактирование и удаление своих сообщений;
- поиск по текущей переписке;
- отображение онлайн-статуса;
- светлая и темная тема.

## Локальный запуск

```bash
npm install
npm run dev
```

По умолчанию frontend обращается к API по адресу `http://localhost:8080`.

Если backend запущен на другом адресе, создайте файл `.env.local`:

```env
VITE_API_URL=http://localhost:8080
```

## Скрипты

```bash
npm run dev      # запуск dev-сервера
npm run build    # проверка TypeScript и production-сборка
npm run lint     # ESLint
npm run preview  # локальный просмотр production-сборки
```

## Связь с backend

Frontend использует:

- HTTP-запросы для авторизации, профиля, друзей, чатов, истории сообщений и загрузки файлов;
- WebSocket `/ws?chat_id={id}&token={jwt}` для сообщений в реальном времени и событий presence.

Адрес WebSocket формируется из `VITE_API_URL`: `http` заменяется на `ws`, `https` - на `wss`.

## Что проверить перед сдачей

- `npm run lint`
- `npm run build`
- корректность `VITE_API_URL` на деплое;
- вход тестовыми аккаунтами в режиме инкогнито;
- отправку текстового сообщения и вложения;
- создание личного и группового чата;
- адаптивность интерфейса на узком экране.
